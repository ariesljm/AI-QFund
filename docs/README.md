# 智能进化公募基金推荐与监控系统 (AI-QFund)

本项目基于"原生 Python + 外部 API + 本地 SQLite"极简架构，彻底抛弃高资源消耗框架。系统定位于"每日信号推荐与虚拟追踪"，通过客观动能捕捉中期趋势，并由大语言模型（LLM）进行宏观排雷与逻辑印证。

---

## 1. 系统运行与调度架构

为避免数据延迟与 API 封禁，彻底脱离盘中交易时间，采用"凌晨全量拉取 + 日间只读面板"模式。

### 1.1 核心工作流 (CRON 定时任务)

| 时间 | 模块 | 说明 |
|------|------|------|
| 02:00 | 数据基座与状态机 | 获取上一交易日收盘后全市场基金复权净值/累计净值与宏观宽基指数，判断大盘环境 |
| 02:30 | 虚拟池监控 | 对 HOLD 状态的历史推荐记录盯盘，触发止损或逻辑证伪则标记 EXIT |
| 03:00 | 顺位推荐引擎 | 运行 LightGBM 特征打分输出 Top 10，调用 LLM 进行顺位否决，输出唯一 HOLD 推荐入库 |
| 月末 23:00 | 双轨进化引擎 | 重训 LightGBM 模型权重，清洗错题本规则 |
| 日间 | Web 展现层 | FastAPI + Streamlit 仅对 SQLite 提供只读读取服务 |

## 2. 数据基座与状态机模块 (`data_foundation.py`)

本模块提供所有底层纯客观特征，彻底屏蔽分红导致的净值断崖干扰。

### 2.1 核心数据获取规范

- **净值标准**：强制使用累计净值 (Cumulative NAV) 或复权净值 (Adjusted NAV)，绝不使用单位净值
- **增量更新**：建立本地 HDF5 或 SQLite 历史数据池，每日仅增量 Fetch T-1 日数据，避免被东方财富等接口源封禁 IP

### 2.2 宏观大盘状态机 (Regime Switch)

提取"沪深 300 指数"或"万得全A"近 60 日均线 (MA60) 状态：

| 状态 | 条件 | 特征权重偏向 |
|------|------|-------------|
| **BULL** (顺势环境) | 指数 > MA60 | 赫斯特指数 (Hurst)、向上捕获率、动量因子 |
| **BEAR** (防御/反弹环境) | 指数 < MA60 | 卡玛比率 (Calmar)、向下捕获率、乖离率 (BIAS，超跌反弹) |

### 2.3 量化特征工程 (X)

- **动态锚定 RBSA**：以最新季报十大重仓股为约束下限，滚动回归基金前三大隐形行业暴露度
- **动量与记忆特征**：60 日 Hurst 指数、近 20 日绝对动量
- **风险调整特征**：卡玛比率（区间年化收益/最大回撤）、区间下行波动率
- **资金面特征**：RBSA 第一大暴露行业对应的 ETF 近 5 日净申赎份额斜率

## 3. 推荐引擎 (`recommend.py`)

采用"理科大脑（筛选）+ 文科大脑（否决）"的漏斗模型。

### 3.1 目标函数 (Model Target Y)

LightGBM 的预测目标 Y 绝不能是绝对收益，必须是风险调整后的超额收益：

```
Y = R_fund(t+20) - R_hs300(t+20)
```

即未来 20 日相对沪深 300 的超额收益 Alpha，防止模型在熊市推荐高波动垃圾基。

### 3.2 LLM 顺位否决机制 (Sequential Veto)

将 LightGBM 打分前 10 名的基金及具体特征，喂给 LLM 执行一票否决。

**Prompt 模板结构：**

1. **【系统状态与规则】** 当前大盘状态、策略侧重、核心宪法（来自进化 JSON）
2. **【今日宏观文本与资金流向】** 财经新闻摘要、领涨/领跌行业、ETF 净流入数据
3. **【候选名单 (按超额收益动能排序)】** 代码、真实重仓行业 (RBSA)、核心量化特征
4. **【任务指令】** 严格从第 1 名开始向下验证；若重仓行业符合宏观资金流向且无政策利空，立即选定并停止；若有明确利空或违背核心宪法，行使否决权并记录理由

## 4. 虚拟池监控引擎 (`monitor.py`)

对所有尚未平仓的 HOLD 基金进行每日扫描，捕捉趋势破裂信号。**绝对不设固定 ROI 止盈。**

### 4.1 量化追踪止损（第一防线）

- 读取基金买入后创下的最高累计净值 (`highest_nav`)
- 若当前净值较最高点回撤超过 **2 倍 ATR** 或动态阈值，强制标记 EXIT

### 4.2 RBSA 风格漂移监测（第二防线）

- 若最新 RBSA 第一大权重行业突降超过 **15%**，判定基金经理主动下车原主线，标记 EXIT

### 4.3 LLM 逻辑证伪（第三防线）

- 提取入库时的 `buy_reason`，结合今日最新新闻
- 若 LLM 判定逻辑链条断裂，发出预警或平仓信号

## 5. 进化引擎 (`evolve.py`)

月末 23:00 触发，从错误中学习。

- 查询当月所有 EXIT 且亏损超过 **-5%** 的交易记录
- 提取 `buy_reason` + `sell_reason`，调用 LLM（Meta-Prompting 视角: 系统优化架构师）
- 要求输出一条硬规则，追加到 `evolution_rules.json`
- 次日推荐引擎自动吸纳扩充的错题记忆

## 8. 架构约束

- **内存隔离**：所有 Cron 命令通过 `ulimit -v` 强制内存上限
- **零驻留**：每个脚本执行完毕即被内核销毁，不常驻内存
- **去中间件化**：原生 requests 直连东方财富/天天基金 API，不用 AkShare 等重型库
- **正则提取**：HTML 表格用编译期正则匹配，避免 BeautifulSoup 等 DOM 解析器的内存开销
